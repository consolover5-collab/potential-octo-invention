# tg-parsing

Бот для мониторинга Telegram-чатов (барахолок): ищет сообщения по ключевым словам/фото и отправляет продавцу DM.

## Что умеет

- Мониторит заданные чаты через Telethon userbot
- Фильтрует по ключевым словам и (опционально) по фото через Groq Vision
- Проверяет лимит цены
- Не отправляет DM повторно одному и тому же продавцу (dedup в SQLite)
- Даёт управление через Telegram-бота (aiogram)

## Быстрый старт (локально)

1. Установите Python 3.11+.
2. Создайте конфиг:

   ```bash
   cp config.example.json config.json
   ```

3. Заполните `config.json`:
   - `telegram.api_id` и `telegram.api_hash` — из https://my.telegram.org
   - `telegram.phone` — ваш номер в формате `+7...`
   - `telegram.bot_token` — токен от `@BotFather`
   - `vision.api_key` — ключ Groq (если используете vision)
4. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

5. Запустите:

   ```bash
   python main.py
   ```

После запуска откройте вашего Telegram-бота и отправьте `/start` для управления.

## Конфигурация

Основные поля в `config.json`:

- `monitoring.chats` — список чатов для мониторинга (`@username` или id)
- `monitoring.keywords` — ключевые слова
- `monitoring.max_price` — максимальная цена (в рублях)
- `monitoring.use_vision` — включить/выключить анализ фото
- `actions.dm_message` — текст DM продавцу
- `rate_limits.dm_per_hour` — лимит DM в час

Полный шаблон — в `config.example.json`.

## 🔑 Как получить ключ Groq (бесплатно)

Vision-анализ фото работает через Groq — бесплатный AI-сервис. Карта не нужна.

### Шаг 1: Регистрация
1. Откройте https://console.groq.com
2. Нажмите **Sign Up**
3. Зарегистрируйтесь через Google, GitHub или email
4. Подтвердите почту, если потребуется

### Шаг 2: Создание API-ключа
1. После входа в консоль откройте раздел **API Keys** (в левом меню)
2. Нажмите **Create API Key**
3. Введите любое название (например, `tg-parsing`)
4. Нажмите **Submit**
5. **Скопируйте ключ сразу!** Он начинается с `gsk_...` и показывается только один раз

### Шаг 3: Вставьте ключ в конфиг
Откройте `config.json` и вставьте ключ:

```json
"vision": {
    "api_key": "gsk_ваш_ключ_сюда"
}
```

### Бесплатные лимиты Groq

| Параметр | Лимит |
|----------|-------|
| Запросов в минуту | 30 |
| Запросов в день | 1 000 |
| Токенов в день | 500 000 |

#### Сколько чатов можно мониторить?

| Чатов | Фото/день примерно | Хватит? |
|:-----:|:-------------------:|:-------:|
| 1–2   | ~50                 | ✅ Легко |
| 3–5   | ~150                | ✅ Да    |
| 5–10  | ~300                | ⚠️ Впритык |
| 10+   | ~500+               | ❌ Нужен платный план |

> 💡 Vision вызывается **только** когда нет совпадения по ключевым словам, но есть фото.
> Хорошо подобранные ключевые слова = меньше вызовов Vision = лимита хватит дольше.
>
> Если лимит исчерпан — бот просто пропускает фото, но продолжает работать по тексту.

## Docker

1. Подготовьте `config.json`, `data/` и `session/` в соответствии с путями из `deploy/docker-compose.yml`.
2. Запустите:

   ```bash
   docker compose -f deploy/docker-compose.yml up -d --build
   ```

## systemd (Linux/VPS)

Есть пример сервиса: `deploy/tg-parsing.service`.

Путь/пользователь в нём нужно адаптировать под вашу систему, затем:

```bash
sudo cp deploy/tg-parsing.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now tg-parsing
```
